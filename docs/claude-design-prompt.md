# Claude Design Prompt — AI 하네스 빌더 UI 재설계

## 프로젝트 개요

AI 에이전트 팀을 드래그 앤 드롭으로 조합하고, 파이프라인을 구성하여 내보내는 비주얼 하네스 빌더입니다.

- **라이브 사이트**: https://ai-harness-builder-mu.vercel.app
- **GitHub**: https://github.com/dragonku/ai-harness-builder
- **기술 스택**: Next.js 16 + React Flow v11 + shadcn/ui + Tailwind CSS + Zustand

## 디자인 시스템

이미 만들어둔 디자인 시스템을 적용해주세요. (Design System은 프로젝트 온보딩 시 자동 감지됩니다)

## 재설계 요청

현재 MVP 수준의 기본 UI를 **현대적인 노드 에디터 UX**로 재설계해주세요.

### 현재 구조 (유지)

```
┌──────────┬─────────────────────────┬──────────────┐
│ 사이드바  │     캔버스 (React Flow)   │  속성 패널    │
│ (w-64)   │     (flex-1)            │  (w-72)      │
├──────────┴─────────────────────────┴──────────────┤
│  툴바 (상단)                                        │
└────────────────────────────────────────────────────┘
```

### 재설계 방향

**참고 레퍼런스** (이런 느낌의 현대적 에디터):
- n8n (워크플로우 에디터)
- Langflow (AI 파이프라인 빌더)
- Figma (캔버스 + 패널 구조)
- Linear (미니멀 + 기능적)

### 화면별 재설계 요청

#### 1. 캔버스 영역
- 캔버스 배경: 도트 그리드 또는 미세한 격자 패턴
- 줌/패닝 컨트롤을 미니맵과 함께 우하단에 배치
- 캔버스가 비어있을 때: 중앙에 CTA ("템플릿에서 시작" / "빈 캔버스에서 시작")

#### 2. 에이전트 노드 (핵심)
현재: 단순 흰색 박스 + 텍스트. 재설계:
- 노드 상단에 컬러 바 (모델별 색상: haiku=green, sonnet=blue, opus=purple)
- 아이콘: 각 role에 맞는 아이콘 (planner=지도, executor=코드, reviewer=돋보기 등)
- 노드 바디: role 이름 (볼드), model 뱃지, 스킬 개수 표시
- 호버 시: 살짝 올라오는 그림자 + 스킬 목록 툴팁
- 선택 시: 테두리 강조 + 미세한 글로우 효과
- 입출력 핸들: 원형, 연결 시 색상 변화

#### 3. 엣지 (연결선)
- 순차(sequential): 실선 + 화살표, 회색/다크
- 병렬(parallel): 점선 + "parallel" 라벨 뱃지, 보라색
- 엣지 호버 시: 하이라이트 + 삭제 버튼 표시

#### 4. 사이드바
- 상단: 앱 로고 + 이름 ("하네스 빌더")
- 탭: "에이전트" / "템플릿" (아이콘 + 텍스트)
- 에이전트 팔레트:
  - 카드형 목록 (아이콘 + role + 한줄 설명 + model 뱃지)
  - 드래그 시: 고스트 이미지로 노드 프리뷰
- 템플릿 갤러리:
  - 카드에 미니 파이프라인 다이어그램 프리뷰
  - 에이전트 수, 단계 수 뱃지
  - "사용하기" 버튼

#### 5. 속성 패널
- 상단: 선택된 에이전트 아이콘 + role 이름
- 섹션 구분: "기본 정보" / "도구" / "스킬"
- Model 선택: 세그먼트 컨트롤 (Haiku | Sonnet | Opus)
- Tools: 토글 칩 (ON/OFF)
- Skills: 접을 수 있는 아코디언, 프롬프트는 코드 에디터 스타일
- 하단: "에이전트 삭제" 버튼 (빨간색, 확인 필요)

#### 6. 툴바
- 좌측: 하네스 이름 (인라인 편집)
- 중앙: (빈 공간 또는 브레드크럼)
- 우측: 저장 / 불러오기 / 내보내기 버튼 그룹
- 내보내기 버튼: 드롭다운으로 포맷 선택 (Claude Code / YAML / JSON / 전체)

#### 7. 내보내기 다이얼로그
- 포맷별 아이콘과 설명
- 검증 오류 시: 인라인 에러 목록 + "문제로 이동" 링크
- 다운로드 진행 표시

#### 8. 빈 상태 / 온보딩
- 첫 방문 시: 중앙에 환영 메시지 + 2개 CTA 카드
  - "템플릿에서 시작하기" — 추천, 프라이머리 색상
  - "빈 캔버스에서 시작하기" — 세컨더리
- 각 CTA 카드에 간단한 일러스트 또는 아이콘

### 추가 고려사항

- **다크 모드 지원**: 라이트/다크 토글 (시스템 설정 따르기 기본)
- **반응형**: 최소 너비 1024px (데스크톱 전용 경고)
- **애니메이션**: 노드 추가/삭제 시 부드러운 트랜지션, 엣지 연결 시 짧은 피드백 애니메이션
- **키보드 단축키**: Delete(노드 삭제), Cmd+S(저장), Cmd+E(내보내기)

### 산출물 요청

1. **메인 에디터 화면** — 템플릿이 로드된 상태 (4개 노드 + 엣지)
2. **빈 캔버스 / 온보딩 화면**
3. **속성 패널 열린 상태** — 노드 선택 시
4. **내보내기 다이얼로그**
5. **다크 모드 버전**

각 화면을 개별 프레임으로 만들어주세요.
